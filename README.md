# Тестовое задание Devmakers

## О проекте

Этот репозиторий содержит выполненное тестовое задание для компании Devmakers. В процессе работы проект был реализован по требованиям и дополнительно значительно улучшен по структуре, производительности и удобству сопровождения.

## Что было сделано

### Архитектура и структура проекта

Проект был переработан с опорой на принципы модульности и разделения ответственности. Каждая часть кода теперь имеет чётко определённую роль:

* **app-pages** — страницы с локальными компонентами, хуками и логикой
* **shared** — общие модули: данные, утилиты, layouts, провайдеры, стили, модалки, SEO, тема
* **ui** — переиспользуемые UI-элементы: атомы, блоки среднего уровня и составные компоненты
* **widgets** — виджеты, объединяющие UI-компоненты и логику

Такой подход устраняет дублирование, облегчает навигацию и делает проект масштабируемым.

### Производительность

Оптимизация проводилась на **реальном мобильном устройстве** и через **Chrome DevTools FPS Meter**. Итог — **значительный рост FPS** и более плавные анимации, особенно на слабых устройствах. Были уменьшены тяжёлые анимации и оптимизированы крупные компоненты.

### Оптимизация UI

UI-слой был переработан и структурирован по уровням сложности:

* атомарные элементы
* блоки среднего уровня
* составные компоненты

Это упростило развитие интерфейса и исключило смешение ответственности.

### Работа с данными

Общие данные и вспомогательные функции были пересмотрены и приведены к единообразию. Проектные данные вынесены в отдельные модули в `shared/data`.

### Улучшение читаемости и поддержки

* переименованы директории и файлы
* логика распределена по слоям
* удалён мёртвый код и лишние зависимости
* подготовлена структура, удобная для внедрения новых страниц и компонентов

## Потенциальные улучшения

Хотя текущая версия полностью рабочая и оптимизированная, есть направления, которые можно развивать дальше:

* перенос части анимаций в чистый CSS для снижения нагрузки на главный поток
